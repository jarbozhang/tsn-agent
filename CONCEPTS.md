# Concepts

本项目的共享领域词汇——具有项目特定含义的实体、命名流程与状态概念。以核心领域词汇播种，随 ce-compound / ce-compound-refresh 处理学习而增长；直接编辑亦可。仅作词汇表，不是规格说明或杂物抽屉。

## Skill 指引与场景

### Skill 指引（Skill Guidance）
随每次 agent 运行注入 system prompt 的可编辑领域指引，承载领域语义、操作流程与推荐参数；用户可在应用内编辑，下次运行生效。与之相对，协议不变量（用户改坏会破坏数据对账的规则）固化在不可编辑的骨架中，不进指引文件。

### 场景（Scenario）
会话所属的行业语境（如通用 TSN、箭载/舰载 TSN），由会话的场景配置 id 标识。场景决定 agent 收到哪份场景 Reference、模板目录过滤后返回哪些候选模板。未知或缺失场景一律按通用场景回退处理。

### 场景 Reference（Scenario Reference）
按场景拆分的指引文件，承载该场景的模板选择规则、推荐参数默认与规范图 preset 表。主索引（场景无关）每次运行必注入；当前场景的 Reference 随之注入，其余场景文件以绝对路径表列出供 agent 按需查阅。

## 出厂生命周期

### 出厂内容（Factory Content）
随应用安装包内置分发的 skill 文件原始版本。是播种的来源、恢复的目标基线；用户编辑产生的内容永远不是出厂内容。

### 播种（Seeding）
应用解析 skill 根时，把出厂内容复制进用户可写数据目录的过程。逐文件三态判定：缺失则补播；内容等于当前出厂则仅登记不重写（防与用户保存竞态）；内容等于某个历代出厂版本则静默升级为新出厂；其余视为用户编辑，保留并标记。个别文件失败不阻断其余文件，下次解析自愈。

### 出厂清单（Factory Manifest）
用户数据目录中记录每个出厂文件真实出厂哈希与用户修改态的隐藏登记文件。它是缓存而非真相源——判定真相每次从文件内容哈希重新推导，清单缺失或损坏时由历代出厂哈希兜底，因此清单写失败不致命。不变量：哈希表只登记真实出厂哈希，用户态用显式标记表达，绝不存伪哈希。

### 历代出厂哈希（Historical Factory Hashes）
应用二进制内嵌的（路径，哈希）对照表，收录每个发布版本的出厂文件哈希。解决「首次升级悖论」：存量安装没有出厂清单时，靠此表判定盘上文件是否为未被编辑的旧出厂内容、能否静默升级。按路径分键——同一内容出现在别的文件路径时不视为出厂内容，防跨文件误判覆写。发版时只追加、永不删除旧条目。

### 出厂移除清单（Factory Removal List）
新出厂布局已删除的文件清单。播种时按保守语义处理孤儿：内容命中该路径的历代出厂哈希则删除，用户改过则保留。

### 恢复内置版本（Factory Restore）
用户显式确认枚举清单后，把全部出厂文件覆写回当前出厂内容、删除出厂移除清单中的文件、重置出厂清单的操作。与播种的保守语义相对：恢复是经确认的破坏性操作，会覆写用户编辑过的出厂文件；仅用户自建文件（不在出厂清单内）保留不动。执行前必先以试运行枚举差异清单供用户确认。
