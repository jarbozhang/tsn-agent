TSN 规划中转服务 API 对接文档

版本：v0.1.0　日期：2026-05-22　适用范围：第一版时间触发/ST 规划闭环。

本文是唯一对外接口文档。调用方按本文完成接口对接；方案背景、部署运维、推进 TODO 不写入本文正文。

接口总览

接口方法用途成功核心返回
/api/start_plan/POST提交完整规划输入并启动规划。plan_id、state
/api/query_plan_status/POST查询任务状态；任务运行中时刷新一次内部规划结果。state、internal_result
/api/get_plan_result/POST任务成功后读取绑定到该任务的源码输出快照。source_outputs、output_fingerprints
/api/stop_plan/POST停止当前正在运行的规划任务。stopped_plan_id

调用顺序

1调用 /api/start_plan/，保存响应中的 data.plan_id。
2轮询 /api/query_plan_status/，直到 data.state=succeeded、failed 或 cancelled。
3仅当 data.state=succeeded 时调用 /api/get_plan_result/。
可选需要主动停止时调用 /api/stop_plan/。该接口停止当前运行任务，不做发起人归属校验。

公共协议

本文示例分为两类：契约示例 表示按当前接口契约构造、用于说明字段和值格式；实测证据 表示来自测试机真实端到端烟测记录。除明确标注为实测证据外，示例中的 plan_id、时间和 hash 均为格式示例。

Base URLhttp://100.78.48.43:18080（测试环境）。
Content-Typeapplication/json; charset=utf-8。
请求包装业务参数统一放在 sendData 中。
HTTP 状态码接口业务响应统一返回 HTTP 200；调用方以 err_code 和 data.state 判断业务结果。未捕获系统异常按 FastAPI 默认错误响应处理，不属于本文业务出参。
时间格式响应中的时间字段使用 ISO-8601，带时区，例如 2026-05-22T14:00:00+08:00。

通用响应体

字段类型必返说明示例
err_codeinteger是业务处理码，0 表示请求已被中转层正常处理。0
err_msgstring是中文业务说明。规划运行中
dataobject是接口业务数据。{"state":"running"}
trace_idstring是单次调用追踪 ID。trace-20260522-140000-000001
timestampstring是响应生成时间。2026-05-22T14:00:00+08:00

POST /api/start_plan/

功能提交完整规划输入，启动一次时间触发/ST 规划。
并发规则同一时间只允许一个规划任务运行。已有运行任务时返回 1003 busy，不排队、不创建新任务。
第一版模式sendData.mode 固定为 time-trigger。

入参字段

顶层字段

字段类型必填取值范围单位说明
sendData.modestring是time-trigger无规划模式。第一版只支持该值。
sendData.source_configobject是固定对象无规划输入配置。
source_config.cfg_parameterobject是固定对象无节点参数配置。
source_config.flow_featurearray是非空数组无流量特征配置。
source_config.topo_featurearray是非空数组无拓扑链路配置。

cfg_parameter 字段

字段类型必填取值范围单位说明
cfg_parameter.cfg_parameter.nodearray是非空；node_id 唯一无节点参数数组。
node[].node_idstring是十进制整数字符串无节点参数 ID。
node[].system_clockstring是十进制数字字符串待确认系统时钟参数。
node[].rc_thresholdstring是正整数字符串待确认RC 流监管阈值。
node[].hpriority_policing_thresholdstring是正整数字符串待确认高优先级 policing 阈值。
node[].lpriority_policing_thresholdstring是正整数字符串待确认低优先级 policing 阈值。
node[].qbv_or_qchstring是"0" 或 "1"无QBv/Qch 相关配置开关。
node[].qci_enablestring是"0"无Qci 开关。第一版只确认 "0"。
node[].ptp_thresholdstring/null否"0"、null 或缺省待确认PTP 阈值；缺省或 null 时不写入源码配置文件。
node[].port_numstring是非负整数字符串个端口数。
node[].node_typestring是"0" 或 "1"无节点类型。"0" 交换机，"1" 网卡。
node[].vlan_cfgstring/null否"0"、null 或缺省无VLAN 配置开关；缺省或 null 时不写入。
node[].vlan_idstring/null否"0"、null 或缺省无VLAN ID；缺省或 null 时不写入。

flow_feature 字段

字段类型必填取值范围单位说明
flow_feature[].stream_idinteger是>= 0；数组内唯一无流 ID。
flow_feature[].src_nodeinteger/null否>= 0无源节点 ID。存在时必须出现在拓扑端点集合中。
flow_feature[].dst_nodeinteger/array/null否拓扑端点节点 ID，或节点 ID 数组无目的节点 ID。
flow_feature[].path_numberinteger/null否>= 1条路径数量参数；第一版不强制等于 path.length。
flow_feature[].sizeinteger是> 0byte报文大小。
flow_feature[].periodnumber是> 0us流量周期。
flow_feature[].patharray是非空数组无候选路径数组。
flow_feature[] 附加字段任意 JSON否源码字段无允许随请求透传，例如 cbs_or_ats、bandwidth 等源码字段。

path 字段

字段类型必填取值范围单位说明
path[].routearray是非空非负整数数组；每项必须存在于 topo_feature[].link_id无链路 ID 序列。
path[].flow_typestring/null否缺省或 ST无流类型。传入时第一版只允许 ST。
path[].latency_requirementnumber是>= 0us延时要求。
path[].jitter_requirementnumber是>= 0us抖动要求。
path[].redundantinteger是0 或 1无冗余标识。
path[].fl_api_flaginteger是0 或 1无源码标识字段。
path[].delay_paranumber是>= 0待确认延时相关参数。
path[].src_ipstring是IPv4 地址无源 IP。
path[].dst_ipstring是IPv4 地址无目的 IP。
path[].src_portinteger是0..65535无源端口。
path[].dst_portinteger是0..65535无目的端口。
path[].dst_macstring是冒号分隔 6 字节十六进制 MAC无目的 MAC。
path[].ip_protocolinteger是0..255无IP 协议号，例如 UDP 为 17。
path[].fivetuple_maskinteger是十进制写法只包含 0/1，长度 1 到 5无五元组掩码。
path[] 附加字段任意 JSON否源码字段无允许随请求透传。

topo_feature 字段

字段类型必填取值范围单位说明
topo_feature[].link_idinteger是>= 0；数组内唯一无链路 ID。
topo_feature[].src_nodeinteger是>= 0无链路起点节点 ID。
topo_feature[].src_portinteger是>= 0无链路起点端口。
topo_feature[].dst_nodeinteger是>= 0无链路终点节点 ID。
topo_feature[].dst_portinteger是>= 0无链路终点端口。
topo_feature[].speednumber是> 0待确认链路速率。
topo_feature[].st_queuesinteger是>= 1个ST 队列数。
topo_feature[].macroticknumber是> 0待确认macrotick 参数。
topo_feature[] 附加字段任意 JSON否源码字段无允许随请求透传。

跨字段校验

唯一性node[].node_id、flow_feature[].stream_id、topo_feature[].link_id 分别要求数组内唯一。
流端点src_node、dst_node 存在时，必须出现在 topo_feature 的端点集合中。
路径链路path[].route[] 的每个链路 ID 必须存在于 topo_feature[].link_id。
空值处理可选字段缺省或显式 null 时，写入源码配置文件会被排除，不落盘为 null。

出参字段

字段类型必返取值范围说明
data.statestring是running / succeeded / failed / busy启动处理后的任务状态。
data.plan_idstring创建任务时是plan-YYYYMMDD-HHMMSS-xxxxxx新任务 ID。
data.started_atstring/null创建任务时是ISO-8601启动时间。
data.running_plan_idstringbusy 时是plan_id当前正在运行的任务 ID。
data.running_duration_msinteger/nullbusy 时是>= 0当前任务运行时长，毫秒。
data.suggestionstringbusy 时是retry_later_or_stop_current调用方处理口径。
data.detailarray参数错误时是Pydantic 错误数组字段校验失败详情。

POST /api/query_plan_status/

功能按 plan_id 查询规划任务状态。任务为 running 时，中转层会刷新一次内部规划结果。

入参字段

字段类型必填取值范围说明
sendData.plan_idstring是非空字符串任务 ID，来自 start_plan 成功响应。

出参字段

字段类型必返取值范围说明
data.plan_idstring是plan_id任务 ID。
data.statestring是running / succeeded / failed / cancel_requested / cancelled / not_found任务状态。
data.running_duration_msinteger/null是>= 0 或 null运行时长，毫秒。
data.started_atstring/null是ISO-8601 或 null启动时间。
data.updated_atstring/null是ISO-8601 或 null最近更新时间。
data.finished_atstring/null是ISO-8601 或 null结束时间。
data.internal_resultinteger/null是0 / 1 / 2 / 其他 / null源码 /api/plan/ 原始结果。0 运行中，1 自动规划完成，2 手动规划完成。
data.error_codeinteger/null是错误码或 null任务失败时记录的错误码。
data.error_messagestring/null是字符串或 null任务失败说明。

POST /api/get_plan_result/

功能任务成功后，返回绑定到该 plan_id 的源码输出快照。
读取条件任务必须为 succeeded，否则返回 1004 plan_not_finished。

入参字段

字段类型必填取值范围说明
sendData.plan_idstring是非空字符串任务 ID。

出参字段

字段类型必返取值范围说明
data.plan_idstring是plan_id任务 ID。
data.statestring是succeeded / running / failed / not_found读取状态。
data.source_outputs.solution_jsonarray/object成功时是源码输出solution.json 内容快照。
data.source_outputs.tsnlight_plan_cfg_jsonobject成功时是源码输出tsnlight_plan_cfg.json 内容快照。
data.output_fingerprints.solution_json.file_namestring成功时是solution.json文件名。
data.output_fingerprints.solution_json.size_bytesinteger成功时是>= 0文件大小，字节。
data.output_fingerprints.solution_json.sha256string成功时是64 位十六进制文件 SHA-256。
data.output_fingerprints.solution_json.mtime_nsinteger成功时是>= 0文件修改时间，纳秒时间戳。
data.output_fingerprints.tsnlight_plan_cfg_json.*object成功时是同上tsnlight_plan_cfg.json 指纹字段。

源码输出关键字段

字段类型单位说明
solution_json[].link_idinteger无链路 ID。
solution_json[].gcl_entriesarray无该链路的门控调度条目数组，可为空。
gcl_entries[].intervalnumber待确认门控持续区间，保持源码值。
gcl_entries[].statestring无门控状态，二进制字符串，例如 0b001。
gcl_entries[].stream_idarray无该条目承载的流 ID 数组，可为空。
tsnlight_plan_cfg_json.network_plan_cfg.nodearray无硬件节点配置数组。

node[].entryarray无寄存器配置数组。
entry[].addrstring无寄存器地址，十六进制字符串。
entry[].datastring/array无寄存器数据，十六进制字符串或十六进制字符串数组。

POST /api/stop_plan/

功能停止当前正在运行的规划任务。
停止语义源码服务只支持停止当前规划。请求中的 plan_id 仅用于审计/回显，不做归属校验。

入参字段

字段类型必填取值范围说明
sendData.plan_idstring/null否任务 ID、null 或缺省调用方期望停止的任务 ID；不影响实际停止对象。

出参字段

字段类型必返取值范围说明

data.requested_plan_idstring/null是任务 ID 或 null请求传入的任务 ID。

状态与错误码

任务状态

state含义
running规划任务正在运行。
succeeded规划完成，且结果快照已绑定。
failed规划失败、停止失败、参数错误或结果绑定异常。
busy已有规划任务运行，本次启动请求未创建新任务。
cancel_requested已请求停止，内部清理尚未最终确认。
cancelled任务已取消。
no_running_plan停止请求到达时没有正在运行的任务。
not_found任务 ID 不存在。

错误码

err_codeerr_msg常见 state触发条件
0请求处理成功视接口而定请求被中转层正常处理。
1001请求参数不合法failed字段缺失、类型错误、枚举不支持或跨字段校验失败。
1002规划任务不存在not_foundplan_id 不存在。
1003当前已有规划任务运行busy已有 active 任务时再次调用 start_plan。
1004规划任务尚未完成running / cancel_requested / failed任务未成功完成时调用 get_plan_result。
1005内部规划服务不可用failed预留错误码。当前内部调用失败主要返回 1006。
1006内部规划失败failed内部规划异常、内部结果失败、输出文件缺失或读取失败。
1007停止规划失败failed内部停止接口异常或任务状态更新失败。
